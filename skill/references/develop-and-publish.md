# 继续开发 + 加界面 + 封装 + GitHub 合规发布(详细脚本)

阶段 5、6 用。核心:让学生用 Agent **真的把项目迭代成可复用作品并合规发布**,不是听讲。

---

## 阶段 5 · 继续开发(动态推荐强化路径 · 硬门槛:必须真的做出来)

核心:**不走固定流水线**。诊断当前项目 → 针对它自由创造性地推荐 2-4 条强化路径 → 学生自选 → 用 Agent 真做出来。

### 第 1 步 · 诊断项目 + 出路径菜单

先判断学生手上这个成果的实际情况(类型 / 能否跑 / 有无界面 / 一次性还是可复用 / 数据从哪来 / 结果够不够好),然后**针对这个具体项目**想出最合适的强化方向,用编号呈现:

```
你这个 [项目] 已经能用了!接下来想往哪个方向做强?我给你几条最适合它的路 👇
1. [针对该项目的强化A] —— 做完你会多一个 [具体好处]
2. [针对该项目的强化B] —— ……
3. [针对该项目的强化C] —— ……
想做哪个?回数字就行(拿不准我推荐 1)。
```

**路径要现想,别套模板。** 下面只是打开思路的例子,**不要局限于此**:
- 🎨 加可视化界面(命令行/纯文字时)
- ➕ 加新功能(功能单一时)
- 🔁 变成可复用工具(一次性时)
- 🔗 接入真实数据源/API(手动喂数据时)
- 🤖 做成自动化:定时/批量(要重复跑时)
- 🎯 优化质量/准确度(结果不够好时)
- 📦 封装成 skill/程序(想反复用时)
- …… 接专业真实场景、多对象对比、生成报告模板、做成分享页 …… 任何更贴合这个项目的

示例(财报翻译机):可推荐「加个网页界面」/「一次对比多家公司」/「接实时行情自动更新」/「封装成一键出报告的工具」。

### 第 2 步 · 学生选定后,用 Agent 实现并跑起来

给带标注模版,让学生把选中的强化丢给他的 Agent。以"加界面"为例(其他路径同理,按所选调整):
```
【你是谁】你是我的开发助手
【背景】我有一个【替换成:你的项目在干什么】的小程序
【帮我做什么】【替换成:你选的强化,如"给它加个网页界面"/"让它能一次对比多家公司"】,
             如果有几种做法,先分析利弊并推荐一个,再动手实现
【怎么给我】我零基础,用大白话;做完把它跑起来,告诉我怎么打开/怎么用
```
若涉及"哪种方案"的选择(如界面用网页还是 Streamlit),**让 Agent 分析推荐、学生拍板**,教练只帮理解利弊。

**⛔ 确认真的做出来了**:学生要能看到强化后的效果(界面能开 / 新功能能跑 / 对比结果出来)。监听共享目录看有没有新文件,或让他贴运行结果。**没跑起来不算完成。**

### 封装成 skill(若学生选了这条,可选进阶)

> "想不想把它变成一个'技能',以后跟你的 Agent 说一句就能自动做?(不做也完全 OK,你的项目已经完整了。)"

仅当学生用 Hermes 且主动想要:引导他让 Agent 用 `skill_manage(action='create', ...)` 建一个**通用**的 skill,`/reload-skills` 后试跑确认生效。

### 硬门槛总则
阶段 5 必须**学生真的完成了所选强化**(能跑 / 能看到效果)。**跳过、只听讲、只看演示——都不算完成,不得进入阶段 6。** 学生想停则软着陆:"到这也是完整一课,想继续随时找我。"

---

## 阶段 6 · 发布到 GitHub(前置:阶段 5 真的完成了)

### 合规预检(必须先做,再碰 git)

**逐条走完这份 checklist,任何一条不过就先解决再发布:**

1. **课程作业?先问。**
   > "先问一句:这个项目是你的课程作业或毕设吗?"
   - 是 → 提醒:公开可能涉及学术诚信/查重风险,**建议先别公开,或去掉可识别信息后再发**,让学生知情后自己决定。
2. **来源合规(二次开发必查)。**
   - 基于某 GitHub 开源项目改的?→ 用 Agent 查原项目 LICENSE:
     - 保留原作者的版权声明和 LICENSE 文件
     - README 注明"本项目基于 [原项目名](链接),遵循 [协议] 协议"
     - MIT/Apache:保留 license 即可;GPL:衍生作品也必须开源
3. **原创声明**:学生/Agent 写的部分,在 README 说明是原创。
4. **数据合规**:项目里不能带爬来的有版权数据、他人隐私数据。有的话先移除。
5. **敏感信息扫描**:让 Agent 扫一遍项目,**杜绝 API key / 密码 / token / 个人隐私**。
   ```
   【帮我做什么】扫描我这个项目文件夹,检查有没有 API key、密码、token、
                个人隐私信息,或任何不该公开的东西,列出来让我确认
   ```
6. **挂 LICENSE**:给自己的项目加一个明确协议(零基础默认推荐 MIT,宽松)。
7. **禁止上传**:抄袭代码、破解/绕过付费的内容、违反平台条款的东西——发现就停,不发。

### 发布流程(用 Agent 一步步做)

给带标注模版,让学生的 Agent 执行:
```
【你是我的】发布助手
【帮我做什么】把当前文件夹的项目发布到我的 GitHub:
  1. 初始化 git、写一个规范的 .gitignore(排除密钥、缓存)
  2. 生成一份清晰的 README(项目介绍/怎么用/致谢原项目)
  3. 加上 MIT LICENSE
  4. 帮我建一个新的 GitHub 仓库并推送上去
【说明】我零基础,每一步先告诉我要做什么再做;需要我登录/授权时提醒我
```

**注意**:
- 学生首次用需要 GitHub 账号 + git 身份配置(user.name/email)+ 登录授权(gh auth 或 token)。Agent 遇到需要授权的步骤要**停下来提示学生操作**,不能替他填凭证。
- 推送成功后,让学生打开自己的仓库链接确认,教练确认无误 → 恭喜他"你有了第一个公开的、署自己名字的 AI 作品"。

### 收尾

> "🎓 你现在的成就:一个能跑、有界面、合规、公开挂在 GitHub 上署你名字的 AI 项目。这已经能写进简历、发给面试官看了。而这一切,你都是靠指挥 Agent 做出来的。"
